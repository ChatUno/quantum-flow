Neural-Link: El Orquestador de Webhooks Forense para Microservicios

No es un simple capturador de datos; es una consola de comando centralizada para interceptar, analizar y rebotar señales entre diferentes servicios web con latencia cero. En un ecosistema fragmentado, 'Neural-Link' es el sistema nervioso que lo conecta todo con precisión quirúrgica.

Descripción Detallada y Funcionalidades Clave:

Terminal de Intercepción (The Neural Gateway):
Una interfaz táctica que visualiza las peticiones entrantes (webhooks) en tiempo real. Cada paquete de datos debe aparecer como una 'incisión' en el dashboard, mostrando el payload JSON con resaltado de sintaxis nativo y tiempos de respuesta de microsegundos.

Simulador de Carga 'Stress-Synapse':
Un motor interno que permite clonar y disparar ráfagas de peticiones de prueba hacia cualquier URL para testear la resistencia de otros sistemas. Debe incluir un monitor de éxito/error con estética de radar militar.

Bóveda de Payloads (The Synaptic Archive):
Utilización de IndexedDB para almacenar miles de registros de peticiones completas, permitiendo al usuario 'viajar en el tiempo' y re-ejecutar cualquier webhook pasado con un solo clic.

Analizador de Latencia 'Pulse-Tracker':
Un sistema que mida el tiempo exacto entre la salida de la señal y la respuesta del servidor remoto, visualizando la eficiencia del flujo en un gráfico de líneas de alta frecuencia (60fps) construido íntegramente en CSS/JS.

Filtro de Ruido 'Surgical-Filter':
Un motor de búsqueda instantáneo manejado por JS que permite filtrar payloads por cabeceras, contenido o código de estado HTTP sin refrescar la interfaz, manteniendo el rendimiento por debajo de los 200ms de respuesta.

Exportador de Esquemas 'Structure-Output':
Función para transformar cualquier JSON interceptado en una estructura de clases de JavaScript o tipos de TypeScript lista para ser inyectada en otros proyectos del imperio.

Público Objetivo:
Desarrolladores de backend, integradores de sistemas y arquitectos de automatización que necesitan ver qué está pasando 'bajo el capó' de sus conexiones en la nube.

Restricciones Técnicas del Butcher:

HTML: 100% Semántico, optimizado para la visualización de grandes volúmenes de texto técnico.

CSS: Arquitectura basada en variables :root con una paleta 'Deep Purple & Neon'. Uso de CSS Grid para un dashboard de múltiples paneles que no colapse en pantallas pequeñas.

JavaScript: Manipulación directa del DOM para actualizaciones en tiempo real. Uso de fetch API avanzado con gestión de timeouts y reintentos automáticos. Cero bloat, cero dependencias.
